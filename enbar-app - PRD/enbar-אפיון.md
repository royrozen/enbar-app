# אפיון — אפליקציית דיווח יומי לצוותי התקנה | ענבר תעשיות פח

**גרסה:** 1.0 | **תאריך:** 12.07.2026 | **איש קשר בלקוח:** אור — or@enbarsteel.com
**סוג מוצר:** Web App מובייל-פירסט, עברית RTL בלבד.

---

## 1. רקע ומטרה

ענבר תעשיות פח (חיפה) מייצרת ומתקינה תעלות מיזוג אוויר באתרי בנייה. כיום דיווחי העבודה היומיים מהאתרים מגיעים בטלפון/וואטסאפ בצורה לא מסודרת, וקשה למנהל לעקוב אחרי התקדמות, בעיות וחריגות שדורשות אישור מפקח. המטרה: אפליקציית ווב פשוטה שבה ראש צוות ממלא דוח יומי מהטלפון תוך 2–3 דקות (כולל תמונות מהשטח), והמנהל רואה את כל הדוחות מכל הפרויקטים במקום אחד — כולל מעקב אחרי חריגות שממתינות לאישור.

---

## 2. משתמשים ותרחישי שימוש

### ראש צוות (מדווח)
- נמצא באתר בנייה, עובד מהטלפון האישי, לפעמים עם כפפות, בשמש חזקה, בעמידה.
- **נגזרות UX:** כפתורים ואזורי מגע גדולים (מינימום 48px), ניגודיות גבוהה, מעט שדות, מינימום הקלדה.
- תרחיש: בסוף יום עבודה פותח את האפליקציה, בוחר את שמו, לוחץ "דוח חדש", ממלא, מצלם/מעלה תמונות, שולח. סיים.

### מנהל (צופה ומאשר)
- יושב במשרד (או מהנייד), רוצה תמונת מצב: מה קרה היום בכל האתרים, אילו בעיות התגלו, ואילו חריגות ממתינות להחלטה.
- תרחיש: פותח דשבורד, מסנן לפי פרויקט/תאריך, נכנס לדוח, רואה תמונות, מסמן חריגה כ"אושר"/"נדחה".
- תרחיש משני: מוסיף פרויקט חדש או ראש צוות חדש למערכת.

**ללא אימות בשלב 1** — ראש צוות בוחר את שמו מרשימה. מסך המנהל בכתובת נפרדת (למשל `/manager`), ללא סיסמה בשלב זה.

## 3. מסכים ופירוט

### 3.1 מסך בחירת שם ראש צוות (`/`)
- לוגו ענבר + כותרת "מי אתה?".
- רשימת ראשי צוותים ככפתורים גדולים (שורה מלאה לכל שם).
- הבחירה נשמרת ב-localStorage — בכניסה הבאה מדלגים ישר למסך הראשי (עם אפשרות "החלף משתמש").

**קריטריוני קבלה:** הרשימה נטענת מה-DB; לחיצה על שם מעבירה למסך הראשי; בכניסה חוזרת מאותו מכשיר לא נדרשת בחירה מחדש.

### 3.2 מסך ראשי לראש צוות (`/home`)
- ברכה: "שלום, {שם}" + כפתור קטן "החלף משתמש".
- כפתור ראשי גדול ובולט: **"+ דוח חדש"**.
- מתחתיו: "הדוחות שלי" — רשימת הדוחות של ראש הצוות, מהחדש לישן (תאריך, שם פרויקט, מספר תמונות, אינדיקציה אם יש חריגה וסטטוס שלה). לחיצה פותחת צפייה בדוח (קריאה בלבד).

**קריטריוני קבלה:** מוצגים רק דוחות של ראש הצוות הנבחר; כפתור "דוח חדש" פותח את הטופס; דוח שנשלח מופיע ברשימה מיד.

### 3.3 טופס דוח יומי (`/report/new`)

| # | שדה | סוג | חובה | ולידציה / הערות |
|---|------|-----|------|------------------|
| 1 | פרויקט | בחירה מרשימה (select) | חובה | רק פרויקטים פעילים |
| 2 | תאריך | date, ברירת מחדל היום | חובה | לא עתידי; ניתן לאחור (דיווח מאוחר) |
| 3 | תיאור עבודה שבוצעה | טקסט חופשי (textarea) | חובה | מינימום 5 תווים |
| 4 | כמה עובדים היו באתר | מספר — כפתורי +/− גדולים + קלט מספרי | חובה | שלם, 1–50 |
| 5 | תמונות מהשטח | העלאת תמונות מרובות (`capture` מהמצלמה או מהגלריה) | רשות | עד 10 תמונות, עד ~10MB כל אחת; דחיסה בצד לקוח לפני העלאה (~1600px); תצוגה מקדימה + מחיקה לפני שליחה |
| 6 | בעיות שהתגלו באתר | טקסט חופשי — **שדה נפרד מתיאור העבודה** | רשות | ריק = "לא התגלו בעיות" |
| 7 | הצעה לתוספת / חריגות לאישור | טקסט חופשי | רשות | אם מולא — הדוח מסומן עם חריגה בסטטוס "ממתין" ומופיע לבדיקת המנהל |

- כפתור שליחה גדול בתחתית, מוצמד (sticky). בזמן העלאה — מצב טעינה + חיווי התקדמות תמונות; מניעת שליחה כפולה.
- לאחר שליחה מוצלחת: מסך אישור "הדוח נשלח ✓" + חזרה למסך הראשי.
- **Nice-to-have:** שמירת טיוטה אוטומטית ל-localStorage (טקסטים בלבד, לא תמונות) — שחזור אם הדפדפן נסגר באמצע.

**קריטריוני קבלה:** אי אפשר לשלוח בלי שדות חובה (הודעת שגיאה בעברית ליד השדה); תמונות עולות ל-Storage ונקשרות לדוח; דוח עם חריגה נוצר עם סטטוס "ממתין"; כשל רשת מציג שגיאה ברורה בלי לאבד את מה שהוקלד.

### 3.4 מסך מנהל — דשבורד (`/manager`)
- **שורת מצב עליונה:** מספר דוחות היום + **badge בולט** "X חריגות ממתינות לאישור" — לחיצה מסננת לחריגות ממתינות בלבד.
- **סינון:** פרויקט, ראש צוות, טווח תאריכים (ברירת מחדל: 7 ימים אחרונים). הסינונים משולבים.
- **רשימת דוחות אחרונים** (חדש→ישן): תאריך, פרויקט, ראש צוות, מספר עובדים, תמונה ממוזערת ראשונה, אייקון אם דווחה בעיה, badge סטטוס חריגה (ממתין/אושר/נדחה) אם קיימת.

**קריטריוני קבלה:** דוח חדש מופיע בדשבורד ללא רענון ידני (או ברענון פשוט); הסינונים עובדים משולבים; ה-badge מציג את המספר הנכון של חריגות ממתינות.

### 3.5 מסך פרטי דוח (מנהל) (`/manager/report/:id`)
- כל שדות הדוח במלואם, כולל גלריית תמונות (תצוגה מוגדלת בלחיצה).
- אם קיימת חריגה: תיבת החלטה עם כפתורים **"אישור" / "דחייה"** + סטטוס נוכחי. ניתן לשנות החלטה.

**קריטריוני קבלה:** כל הנתונים והתמונות מוצגים; שינוי סטטוס חריגה נשמר ב-DB ומשתקף מיד בדשבורד וברשימת הדוחות של ראש הצוות.

### 3.6 ניהול בסיסי (`/manager/settings`)
- **פרויקטים:** הוספה (שם + כתובת/עיר, רשות), סימון פעיל/לא פעיל (לא פעיל לא מופיע בטופס הדוח; דוחות ישנים נשמרים).
- **ראשי צוותים:** הוספה (שם), סימון פעיל/לא פעיל.
- ללא מחיקה קשיחה בשלב 1 — רק השבתה.

**קריטריוני קבלה:** פרויקט/ראש צוות חדש מופיעים ברשימות הבחירה מיד; מושבת נעלם מהרשימות אך ההיסטוריה נשמרת.

## 4. מודל נתונים (Supabase / Postgres)

```sql
team_leads (
  id          uuid PK default gen_random_uuid(),
  name        text NOT NULL,
  is_active   boolean NOT NULL default true,
  created_at  timestamptz NOT NULL default now()
)

projects (
  id          uuid PK default gen_random_uuid(),
  name        text NOT NULL,
  city        text,                      -- רשות
  is_active   boolean NOT NULL default true,
  created_at  timestamptz NOT NULL default now()
)

reports (
  id                 uuid PK default gen_random_uuid(),
  team_lead_id       uuid NOT NULL REFERENCES team_leads(id),
  project_id         uuid NOT NULL REFERENCES projects(id),
  report_date        date NOT NULL,
  work_description   text NOT NULL,      -- תיאור עבודה שבוצעה
  workers_count      int  NOT NULL CHECK (workers_count BETWEEN 1 AND 50),
  issues             text,               -- בעיות שהתגלו באתר (נפרד!)
  extras_description text,               -- הצעה לתוספת / חריגות לאישור
  extras_status      text CHECK (extras_status IN ('pending','approved','rejected')),
                     -- NULL אם אין חריגה; 'pending' בעת יצירה עם חריגה
  created_at         timestamptz NOT NULL default now()
)

report_photos (
  id           uuid PK default gen_random_uuid(),
  report_id    uuid NOT NULL REFERENCES reports(id) ON DELETE CASCADE,
  storage_path text NOT NULL,            -- נתיב ב-Supabase Storage
  sort_order   int  NOT NULL default 0,
  created_at   timestamptz NOT NULL default now()
)
```

- **Storage:** bucket בשם `report-photos`, נתיב `reports/{report_id}/{uuid}.jpg`. Bucket ציבורי לקריאה בשלב 1 (אין אימות).
- **RLS:** מכיוון שאין אימות בשלב 1, גישה דרך ה-anon key עם policies פתוחות ל-insert/select (ו-update רק ל-`extras_status`). לתעד כחוב טכני לשלב 2.
- תצוגת סטטוס בעברית: `pending` = ממתין, `approved` = אושר, `rejected` = נדחה.

## 5. החלטות UX

- **RTL מלא:** `dir="rtl"` על ה-root, כל הפריסות והאייקונים בכיוון נכון. עברית בלבד — אין i18n.
- **Mobile-first:** עיצוב ל-390px ומעלה; הדשבורד מתרחב יפה לדסקטופ.
- **אזורי מגע גדולים:** כפתורים בגובה 48–56px, פונט קלט 16px+ (מונע זום אוטומטי ב-iOS), ניגודיות גבוהה לעבודה בשמש.
- **טפסים קצרים:** טופס אחד, עמוד אחד, 7 שדות, 3 מהם רשות. מספר עובדים עם +/− במקום הקלדה.
- **שמירת טיוטה (nice-to-have):** localStorage לטקסטים בלבד.
- פונט עברי קריא (Heebo/Assistant), תאריכים בפורמט DD/MM/YYYY.

## 6. טכנולוגיה

| רכיב | בחירה |
|------|--------|
| Frontend | React + Vite |
| עיצוב | Tailwind CSS (מוגדר RTL) |
| DB + Storage | Supabase (Postgres + Storage), גישה ישירה עם supabase-js ו-anon key |
| ניתוב | React Router (`/`, `/home`, `/report/new`, `/manager`, `/manager/report/:id`, `/manager/settings`) |
| אחסון ואירוח | Vercel |
| דחיסת תמונות | בצד לקוח (canvas / browser-image-compression) לפני העלאה |

אין backend ייעודי בשלב 1 — הקליינט מדבר ישירות עם Supabase.

## 7. מחוץ לתחולה בשלב 1

- אימות משתמשים וסיסמאות (כולל למסך המנהל)
- התראות push / מייל / SMS
- ייצוא PDF או אקסל
- אפליקציה נייטיבית (iOS/Android)
- מצב offline מלא (רק טיוטת localStorage בסיסית)
- דיווח שעות/נוכחות עובדים פרטני, חתימות מפקח, ריבוי שפות

## 8. הגדרת "גמור" (Definition of Done)

1. ראש צוות בשטח מגיש דוח מלא עם 5 תמונות מהטלפון בפחות מ-3 דקות, והוא מופיע מיד בדשבורד המנהל.
2. מנהל מסנן דוחות לפי פרויקט + תאריך, פותח דוח, צופה בתמונות, ומאשר/דוחה חריגה — והסטטוס נשמר.
3. מנהל מוסיף פרויקט וראש צוות חדשים והם זמינים מיד בטפסים.
4. האפליקציה פרוסה ב-Vercel, מחוברת ל-Supabase, ועובדת בכרום/ספארי מובייל בעברית RTL תקינה.
